# Tennis-app ChatGPT開発ルール

このファイルは、ChatGPTとの新しいチャットへ移行した場合にも開発方針を確実に引き継ぐための基準です。

## GitHubへの反映方針

- ChatGPTがGitHub連携を利用できる場合、原則として `KEIIZ-3/tennis-app` の `main` ブランチへ直接変更を反映する。
- ユーザーによる全文置換、`git add`、`git commit`、`git push` は、GitHub連携が利用できない場合またはユーザーが明示的に希望した場合だけ案内する。
- 直接変更前に、必ずGitHub上の現行ファイル、既存パス、モデル、ビュー、テンプレート、ログを確認する。
- 既存ファイル名を勝手に変更しない。
- 変更後は、変更ファイル、コミットSHA、確認事項、Render側で必要な操作を報告する。
- Renderの自動デプロイが有効な場合、ユーザーの通常作業は本番画面確認だけとする。

## コード変更ルール

- ユーザーへコードを提示する必要がある場合は、差分ではなく対象ファイルの全文置換形式で提示する。
- パスは必ず `tennis-app/` 起点で正確に記載する。
- 現物コード、ログ、スクリーンショットを優先し、推測だけで既存構成を変更しない。
- 原則として1回の変更は4ファイル以内に抑える。
- スマートフォン表示を優先して設計する。
- 新しいDB項目が必要な場合は、モデル変更とマイグレーションを同じ変更単位に含める。
- 本番データを破壊する可能性がある変更は行わず、既存データとの互換性を保つ。

## 月次精算の確定方針

- 会社は利益を保持する主体ではなく、月次売上を一時保管して分配する「財布」として扱う。
- 会社の内部留保は0円を前提とし、会社残高をマイナスにしない。
- 支払原資は当月売上のみとする。
- 会社売上は、当月のチケット消費金額、回収済み現金売上、ガット張り売上を基本とする。
- 月途中でコーチへの分配を行わず、月末または翌月初に一括精算する。
- コート代は原則として、そのコートを使用した担当メインコーチが負担する。
- 複数のメインコーチが担当した場合は、担当者間で均等負担する。
- 業務委託コーチのみで実施したレッスンのコート代と業務委託報酬は、メインコーチ3人で均等負担する。
- コート代を一括支払いしたコーチには立替額を戻し、各コーチの本来負担額を月末精算で差し引く。
- 雨天中止や返金済みのコート代は最終負担へ含めない。
- コート料金は平日900円/時間、土日祝1,200円/時間、夜間照明400円/時間とする。
- 夜間照明開始時刻は、3月1日〜4月30日および9月1日〜9月30日は18時、5月1日〜8月31日は19時、10月1日〜2月末は17時とする。
- 通常レッスンは2時間として計算する。

## 新しいチャットでの開始手順

1. このファイルを最初に確認する。
2. GitHubの `main` ブランチと直近コミットを確認する。
3. 対象機能の現行ファイルを取得する。
4. 可能であればGitHubへ直接コミットする。
5. Renderのデプロイと本番画面の確認事項をユーザーへ伝える。
